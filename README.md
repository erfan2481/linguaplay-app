# LinguaPlay — ساخت APK با Capacitor + GitHub Actions

این پروژه با **Capacitor** ساخته شده — هیچ محدودیت دامنه یا شبکه‌ای نداره (برخلاف
Appmint)، پس مشکل «پخش صدای آنلاین ناموفق» و باز شدن لینک‌های بیرونی داخل اپ،
اینجا وجود نخواهد داشت.

## قدم‌ها (فقط یه بار لازمه)

### ۱. ساخت ریپازیتوری روی گیت‌هاب
1. وارد github.com بشو (اگه اکانت نداری، رایگان بساز).
2. یه ریپازیتوری جدید بساز (دکمه‌ی سبز "New") — مثلاً به اسم `linguaplay-app`.
   می‌تونه **Private** باشه، مشکلی نداره.
3. تمام فایل‌ها و پوشه‌های همین زیپ رو (با همون ساختار پوشه‌ای) داخل ریپازیتوری
   آپلود کن. ساده‌ترین راه: تو صفحه‌ی ریپازیتوری روی "Add file" → "Upload files"
   بزن و کل محتوای این پوشه رو بکش و ول کن (drag & drop) — گیت‌هاب پوشه‌ها رو هم
   خودش می‌سازه.
   ⚠️ حتماً پوشه‌ی مخفی `.github` هم باید آپلود بشه (شامل فایل build-apk.yml
   هست) — بدون این فایل، ساخت خودکار APK کار نمی‌کنه.

### ۲. اجرای خودکار ساخت APK
- به محض آپلود فایل‌ها روی شاخه‌ی `main`، گیت‌هاب خودش شروع می‌کنه به ساخت APK.
- برای دیدن وضعیتش: تب **Actions** بالای صفحه‌ی ریپازیتوری رو بزن.
- یه اجرا (run) با اسم "Build APK" می‌بینی؛ روش بزن و صبر کن تا تیک سبز بخوره
  (حدود ۵ تا ۱۰ دقیقه طول می‌کشه).

### ۳. دانلود APK
- بعد از اتمام (تیک سبز)، پایین همون صفحه‌ی اجرا، بخش **Artifacts** رو ببین.
- یه فایل zip به اسم `linguaplay-debug-apk` هست — دانلودش کن، بازش کن، داخلش
  `app-debug.apk` هست.
- این فایل رو منتقل کن روی گوشیت و نصبش کن (شاید لازم باشه "نصب از منابع
  ناشناس" رو فعال کنی).

## نکته‌ی مهم: این فقط نسخه‌ی تست (debug) ـه
فایلی که این‌طوری ساخته می‌شه **امضا نشده و برای تست شخصیه**، نه برای انتشار توی
بازار (Cafe Bazaar). برای نسخه‌ی نهایی و قابل‌انتشار (release + امضا شده)، یه
قدم اضافه لازمه (ساخت keystore + تنظیم امضا در گیت‌هاب) — هروقت به اونجا رسیدیم
بگو "بعدی" تا اون بخش رو هم اضافه کنم.

## تغییر اسم پکیج (اختیاری)
داخل فایل `capacitor.config.json`، مقدار `appId` الان `com.jack.linguaplay`ـه.
این یه شناسه‌ی یکتاست که بازار باهاش اپت رو می‌شناسه — اگه دوست داری چیز دیگه‌ای
باشه (مثلاً بر اساس اسم واقعی اپت)، همینجا عوضش کن، قبل از publish نهایی.
